Три подхода к реализации

Выбери подход — это определит баланс между визуальной сложностью и производительностью

WEBGL

A — Three.js WebGL Globe

Фотореалистичный 3D глобус с настоящими орбитальными траекториями. Текстура Земли, атмосферное свечение, анимированные спутники.

За

  • Максимальный wow-эффект
  • Настоящие орбиты TLE
  • Closest to Cloudflare ref

Против

  • ~200KB JS (Three.js)
  • Нужна оптимизация под Lighthouse
  • Lazy load обязателен

Lighthouse: 90–95 при правильной реализации

CANVAS

B — Canvas 2D Globe

Глобус на Canvas API: wireframe-сетка, анимированные орбиты, спутники. Нет зависимости от Three.js. Всё рисуется вручную.

За

  • ~0 внешних зависимостей
  • Полный контроль
  • Лёгкий (~15KB кода)

Против

  • Менее реалистично
  • Больше ручной работы
  • Без текстуры Земли

Lighthouse: 95–100

CSS

C — CSS/SVG Globe

Глобус полностью на CSS transforms + SVG. Орбиты — анимированные SVG-эллипсы. Нет JS для глобуса вообще.

За

  • Zero JS overhead
  • Идеальный Lighthouse
  • GPU-ускоренные анимации

Против

  • Наименее реалистично
  • Ограниченная интерактивность
  • Сложно имитировать 3D

Lighthouse: 98–100

Моя рекомендация: A (Three.js) — при правильной реализации (dynamic import, defer, intersection observer) даёт 90+ Lighthouse и максимальный wow-эффект, как на Cloudflare. Подходит для aerospace-компании.
