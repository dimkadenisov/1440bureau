# Бюро 1440 — Редизайн лендинга

## Цель

Улучшенная версия https://1440.space/ — более привлекательный дизайн, scroll-анимации в стиле Apple, интерактивный 3D-глобус с орбитальными траекториями спутников вместо статичного изображения. Целевой Lighthouse score: 90+.

---

## Стек

| Технология | Роль |
|---|---|
| **Astro** | Фреймворк (статическая генерация, 0KB JS по умолчанию) |
| **TypeScript** | Типизация |
| **Three.js** | 3D-глобус (dynamic import, только при входе в viewport) |
| **GSAP + ScrollTrigger** | Scroll-анимации (lazy, только при входе) |
| **Vanilla CSS** | Стили, CSS custom properties |

**Глобус (Three.js — GitHub-стиль):**
- ~12 000 `CircleBufferGeometry` + `InstancedMesh` для отрисовки суши (без текстуры)
- Карта суши: PNG world map → `canvas.getImageData()` → фильтрация земля/вода
- Орбитальные дуги: `CubicBezierCurve3` + `TubeBufferGeometry` + `setDrawRange()` для анимации
  - Данные: реальные TLE-орбиты спутников «Рассвет-1» и «Рассвет-2» (6 спутников) из Space-Track.org / CelesTrak. Fallback если недоступны: симулированные орбиты с параметрами 550 км/800 км, наклонение ~97.5°
  - 3 варианта орбитальных путей по высоте дуги (как у GitHub)
- Атмосферное свечение: кастомный шейдер на увеличенной сфере за глобусом
- FPS-деградация: 4 уровня, снижает плотность геометрии при < 55.5 FPS за 50 кадров
- SVG-заглушка: мгновенно рендерится, crossfade к WebGL через Web Animations API
- Загрузка: `dynamic import()` только при `IntersectionObserver` (globus входит в viewport)

## Lighthouse 90+ стратегия

1. **Three.js** — `dynamic import()` только при IntersectionObserver. До загрузки — SVG-заглушка.
2. **GSAP** — dynamic import только при первом скролле.
3. **Astro** — 0KB JS по умолчанию для статических компонентов.
4. **Шрифты** — системный стек (`system-ui, -apple-system, sans-serif`) или один woff2 с `font-display: swap`.
5. **Изображения** — WebP/AVIF, lazy loading, явные width/height.
6. **SVG-заглушка глобуса** — обеспечивает мгновенный LCP.
7. **CSS** — нет тяжёлых фреймворков (Tailwind не нужен, хватит CSS custom properties).

## i18n

Astro встроенный i18n (`i18n.defaultLocale: 'ru'`). Два языка: `ru` (по умолчанию) и `en`.

- URL-структура: `/` → русская версия, `/en/` → английская
- Все строки вынесены в `src/i18n/ru.ts` и `src/i18n/en.ts`
- Переключатель в Nav меняет locale и сохраняет текущий путь
- Английский контент — перевод всех текстов оригинала

Структура файлов дополняется:
```
src/
  i18n/
    ru.ts    # все строки на русском
    en.ts    # все строки на английском
  pages/
    index.astro      # /  (ru)
    en/
      index.astro    # /en/ (en)
```
